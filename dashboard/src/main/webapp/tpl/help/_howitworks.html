<#macro howitworks>

Step 1: Create Your Project

As a ministrator, you can create a project and set basic information for it.

Everything starts from here and that's all you need to set up your automation in NATL!

Step 2: Load Test Cases

NATL checks out code repository as defined in the project properties, and compile on all test slaves.

This might take two or more minutes according to project complexity. Once this is done, you should be able to see all test cases, and test groups defined by TestNG annotation.

Step 3: Create An Execution

Simply click "Run Selected" button and everything will be put into the queue for execution.

You can run any test cases available on Test Case, or Test Group. Also, a Test Suite is prefered if same subset of cases are going to be executed for multiple times.

Step 4: Monitor The Execution

You can check the results without waiting for the entire suite being executed.

The charts and logs help you identify regressions at earliest.

Step 5: Check Test Summaries

NATL enables you view historical execution results from different dimensions as you might need.

It saves your execution time and issue diagnosis efforts.

Let's Start!
